Elasticsearch中的所有的查询都会触发相关度得分的计算。对于那些我们不需要相关度得分的场景下，Elasticsearch以过滤器的形式提供了另一种查询功能。过滤器在概念上类似于查询，但是它们有非常快的执行速度，这种快的执行速度主要有以下两个原因：

过滤器不会计算相关度的得分，所以它们在计算上更快一些
过滤器可以被缓存到内存中，这使得在重复的搜索查询上，其要比相应的查询快出许多。

* 范围过滤器
```json
get bank/_search
{
    "query": {
        "bool" : {
            "must" : {
                "match_all": {}
            },
            "filter" : {
                "range": {
                    "balance": {
                        "gte": 1000,
                        "lte": 2000
                    }
                }
            }
        }
    }
}
```
