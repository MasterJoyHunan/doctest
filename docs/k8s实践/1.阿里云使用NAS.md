### k8s 挂载阿里云 nas

首先申请阿里云 nas, nas 按量计费，没有占用空间不收钱，申请的是容量型 nas，此过程略

#### 1）通过阿里云CSI存储组件

此方式需要将自建 k8s 注册为阿里云 ack，相对较为麻烦，没测试过

#### 2）在每台 k8s 机器上挂载 nas

阿里云推荐模式

优点：操作简单

缺点：需要每台电脑都挂载 nas 云盘，每个 pod 都需要创建对应的目录

登录自建Kubernetes集群节点，每台节点配置如下

```sh
cat <<EOF >> /etc/nfsmount.conf
[ Server "file-system-id.region.nas.aliyuncs.com" ] # nas 挂载点，需要在 nas 控制台查看
vers=3
Proto=tcp
Lock=False
resvport=False
rsize=1048576
wsize=1048576
hard=True
timeo=600
retrans=2
EOF
```

安装 nfs 挂载工具

```sh
yum install nfs-utils
```

将 nas 挂载至主机

```
mount -t nfs file-system-id.region.nas.aliyuncs.com:/ /mnt 
```

配置自动挂载，修改 `/etc/fstab` 文件，加入挂载配置

```
file-system-id.region.nas.aliyuncs.com:/ /mnt nfs vers=3,nolock,proto=tcp,rsize=1048576,wsize=1048576,hard,timeo=600,retrans=2,_netdev,noresvport 0 0
```

k8s 使用 nas

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  selector:
    matchLabels:
      app: nginx
  replicas: 1
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.14.2
        ports:
        - containerPort: 80
        volumeMounts:
        - mountPath: /data
          name: test-nfs
      volumes:
      - name: test-nfs
        nfs:
          server: file-system-id.region.nas.aliyuncs.com    # 阿里云NAS文件系统挂载点地址，请根据实际值替换。例如，7bexxxxxx-xxxx.ap-southeast-1.nas.aliyuncs.com。
          path: /    # NAS文件系统目录路径。该目录必须为已经存在的目录或根目录。通用型NAS的根目录为“/”，极速型NAS的根目录为“/share”。
```

#### 3）通过 CSI 插件挂载 nas

> CSI插件为非阿里云CSI组件，阿里云不会对该插件的版本、安全性、准确性进行任何保证，在使用过程中阿里云不对由此引发的任何损害承担责任。请您根据实际情况进行选择合适的方式挂载访问NAS。

优点：可以正常的使用 storage class + pvc 资源，无需每台机器都挂载 nas
